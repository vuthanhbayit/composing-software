When to Use Functional Mixins
You should always use the simplest possible abstraction to solve the problem you’re working on.
Start with a pure function. If you need an object with persistent state, try a factory function. If you
need to build more complex objects, try functional mixins.
Here are some good use-cases for functional mixins:
• Application state management, e.g., something similar to a Redux store.
• Certain cross-cutting concerns and services, e.g., a centralized logger.
• Composable functional data types, e.g., the JavaScript

Caveats
Most problems can be elegantly solved using pure functions. The same is not true of functional
mixins. Like class inheritance, functional mixins can cause problems of their own. In fact, it’s possible
to faithfully reproduce all of the features and problems of class inheritance using functional mixins.
