7. Use parallel code for parallel concepts.
“…parallel construction requires that expressions of similar content and function should
be outwardly similar. The likeness of form enables the reader to recognize more readily
the likeness of content and function.”
∼
William Strunk, Jr., The Elements of Style
There are few problems in applications that have not been solved before. We end up doing the same
things over and over again. When that happens, it’s an opportunity for abstraction. Identify the
parts that are the same, and build an abstraction that allows you to supply only the parts that are
different. This is exactly what libraries and frameworks do for us.
UI components are a great example. Less than a decade ago, it was common to mingle UI updates
using jQuery with application logic and network I/O. Then people began to realize that we could
apply MVC to web apps on the client-side, and people began to separate models from UI update
logic.
