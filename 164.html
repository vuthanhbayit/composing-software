A change in the compiler default settings could break your app, even if you didn’t change any of
your own code. That gotcha bears repeating:
Warning:
Refactoring from a class to an arrow function factory might seem to work with
a compiler, but if the code compiles the factory to a native arrow function, your app will
break because you can’t use
new
with arrow functions.
