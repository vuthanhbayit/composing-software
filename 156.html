Functional Mixins & Functional Programming
“Functional” in the context of functional mixins does not always have the same purity connotations
as “functional programming”. Functional mixins are commonly used in OOP style, complete with
side-effects. Many functional mixins will alter the object argument you pass to them. Caveat emptor.
By the same token, some developers prefer a functional programming style, and will not maintain
an identity reference to the object you pass in. You should code your mixins and the code that uses
them assuming a random mix of both styles.
