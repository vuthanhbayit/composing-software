In other words, the signal flow might look something like this. Imagine the arrows are wires between

1
[ Source ] -> [ Mic ] -> [ Filter ] -> [ Mixer ] -> [ Recording ]
In more general terms, you could express it like this:
1
[ Enumerator ] -> [ Transducer ] -> [ Transducer ] -> [ Accumulator ]
If you’ve ever used music production software, this might remind you of a chain of audio effects.
That’s a good intuition to have when you’re thinking about transducers, but they can be applied
much more generally to numbers, objects, animation frames, 3d models, or anything else you can
represent in software.
